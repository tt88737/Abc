# 看板菜单重整设计

## 背景

当前首页顶部菜单有 10 个入口：总控台、世界杯分析、历史规律观察、推荐跟踪、闯三关、三中三公式、5期窗口、三中三5期窗口、高级分析、手动采集。

这些菜单存在明显重复：

- 历史规律观察、推荐跟踪、5期窗口、三中三5期窗口、高级分析都属于复盘或规律验证。
- 三中三公式、三中三5期窗口都服务三中三决策，但入口分散。
- 手动采集属于数据维护，不应和日常决策入口同级。
- 用户日常真正高频的决策入口是总控台、三中三、闯三关、世界杯。

本次重整目标不是删除能力，而是降低顶部导航复杂度，让一级菜单只保留日常决策入口，把低频验证和维护能力收纳到聚合页。

## 目标

1. 顶部一级菜单从 10 个减少到 5 个。
2. 日常决策入口前置：总控台、三中三推荐、闯三关判断、世界杯比分。
3. 复盘、窗口、规律验证、采集合并到“数据与复盘”聚合页。
4. 保留旧功能，不删除算法、不删除页面，只调整入口层级。
5. 首页跳转按钮继续能进入对应核心功能。

## 新一级菜单

| 新菜单 | 对应 tab | 页面职责 |
|---|---|---|
| 总控台 | decisionHome | 今日重点、数据健康、复盘摘要、风险提醒 |
| 三中三推荐 | threeFormulaGate | 下期 6 码复式、单式组合、当前公式组、近期表现 |
| 闯三关判断 | gateChallenge | 当前关卡、是否开第一关/续第二关/冲第三关 |
| 世界杯比分 | worldcupAnalysis | 今日可跟踪比赛、比分推荐、完赛复盘 |
| 数据与复盘 | dataReview | 低频复盘、窗口观察、高级分析、手动采集入口 |

## “数据与复盘”聚合页

新增 `dataReview` 聚合页，采用入口卡片形式，不把所有表格直接堆到首页。

聚合页卡片如下：

| 卡片 | 跳转 tab | 定位 |
|---|---|---|
| 历史规律观察 | historyPattern | 查看长期规律和历史分布 |
| 推荐跟踪 | recommendationTrack | 跟踪既有推荐在开奖后的表现 |
| 5期窗口 | window5 | 查看特别号 5 期窗口覆盖表现 |
| 三中三5期窗口 | threeWindow5 | 查看三中三复式池窗口表现 |
| 高级分析 | patternWatch | 查看更复杂的统计分析 |
| 手动采集 | manualFetch | 触发或辅助数据采集 |

## 交互规则

1. 顶部导航只展示 5 个一级菜单。
2. 用户点击“数据与复盘”后展示 6 个功能卡片。
3. 功能卡片点击后复用现有 `switchTab(tab)` 进入旧 tab 页面。
4. 旧 tab 不出现在顶部导航，但仍保留 renderer 和 data loader。
5. 顶部 active 状态只对一级菜单高亮；从聚合页进入旧 tab 时，`数据与复盘` 保持高亮。
6. 加载失败仍使用现有加载失败面板，不新增复杂错误流。

## 文案调整

| 旧菜单 | 新入口文案 |
|---|---|
| 世界杯分析 | 世界杯比分 |
| 三中三公式 | 三中三推荐 |
| 闯三关 | 闯三关判断 |
| 手动采集 | 数据与复盘内的“手动采集” |

## 非目标

- 不修改三中三公式算法。
- 不修改闯三关状态机。
- 不修改世界杯比分预测算法或采集源。
- 不删除任何已有低频功能。
- 不引入新的前端框架或路由系统。

## 验收标准

1. `index.html` 顶部只出现 5 个一级菜单。
2. 顶部菜单包含：总控台、三中三推荐、闯三关判断、世界杯比分、数据与复盘。
3. 顶部菜单不再直接出现：历史规律观察、推荐跟踪、5期窗口、三中三5期窗口、高级分析、手动采集。
4. `数据与复盘` 页面能看到上述 6 个功能卡片。
5. 6 个功能卡片都能跳转到原功能页面。
6. 总控台三张重点卡仍能跳转到三中三、闯三关、世界杯。
7. 现有 HTML 结构测试和世界杯/三中三相关测试通过。

## 风险与处理

| 风险 | 处理 |
|---|---|
| 老用户找不到历史功能 | 在“数据与复盘”中保留原功能名称和简短说明 |
| 隐藏旧 tab 后 active 状态混乱 | 增加 tab 到一级菜单的映射，旧 tab 归属到 dataReview |
| 首页跳转失效 | 保留原 tab key 和 renderer，不改业务函数 |
| 聚合页变成新一层复杂页面 | 只做入口卡片，不展开明细表 |

## 实施建议

分两步实施：

1. 先改导航和聚合页，不改各专项页内部内容。
2. 后续再把三中三和闯三关页内部进一步做信息层级收敛。
