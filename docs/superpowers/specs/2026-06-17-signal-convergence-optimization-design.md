# 功能信号收敛优化设计

## 1. 背景

当前项目已经完成菜单收敛，顶部入口从 10 个减少到 5 个。但功能内部仍然存在“结果很多、信号不集中”的问题：

- 首页三张重点卡都能进入专项页，但没有统一告诉用户“今天到底值不值得观察”。
- 三中三直接给出 6 码，缺少强弱判断，容易让弱信号也像推荐。
- 闯三关能显示当前关卡，但“不开第一关 / 暂停”的条件不够前置。
- 世界杯有可跟踪、观察、回避，但首页没有给出总判断。
- 数据与复盘已经收纳低频功能，但首页仍缺少明确的降级口径。

本次优化目标是把项目从“多功能入口”进一步收敛为“高信号决策工具”。

## 2. 核心目标

1. 建立统一的“观察信号等级”。
2. 首页第一屏给出“今日总建议”。
3. 三中三、闯三关、世界杯三类核心模块都按同一套等级展示。
4. 弱信号和无信号不再伪装成推荐，明确降级为观察、复盘或暂停。
5. 不改底层公式和采集逻辑，先做产品判断层。

## 3. 统一信号等级

| 等级 | 含义 | 首页表现 | 用户动作 |
|---|---|---|---|
| 强观察 | 数据新鲜，近期表现达标，有明确下一步 | 高亮，进入专项页 | 可以重点查看 |
| 弱观察 | 有候选信号，但近期表现或风险不足 | 普通显示 | 只做谨慎观察 |
| 仅复盘 | 当前没有可跟踪价值，但历史表现可看 | 降权显示 | 进入数据与复盘 |
| 暂停 | 数据异常、条件不足或连续失效 | 红色/灰色提示 | 不进入推荐判断 |

## 4. 今日总建议

首页 `今日重点` 上方新增一个总建议区，规则如下：

| 三类模块状态 | 今日总建议 |
|---|---|
| 任一模块为强观察 | 今日优先观察强信号模块 |
| 无强观察，但有弱观察 | 今日仅弱观察，不做强推荐 |
| 全部仅复盘 | 今日无可观察信号，建议复盘 |
| 任一核心数据异常且无强观察 | 数据异常，暂停判断 |

总建议文案示例：

- `今日优先观察：三中三推荐`
- `今日仅弱观察：闯三关、世界杯`
- `今日无强信号，建议只看复盘`
- `数据异常，暂停推荐判断`

## 5. 三中三信号规则

三中三先不改公式，只基于现有 `threeFormulaOptimizedRecommendation` 输出做判断。

| 条件 | 等级 |
|---|---|
| 数据未加载或公式计算失败 | 暂停 |
| 近 10 期命中率 >= 30%，且当前漏小于最大连挂 | 强观察 |
| 近 10 期命中率 > 0%，或有完整 6 码输出 | 弱观察 |
| 近 10 期命中 0 / 10，但历史仍可计算 | 仅复盘 |
| 当前漏达到或超过历史最大连挂 | 暂停 |

首页卡片展示：

- 标题：`三中三推荐`
- 等级：`强观察 / 弱观察 / 仅复盘 / 暂停`
- 主文案：`下期 6 码：xx xx xx xx xx xx`
- 说明：`近 10 期命中 x / 10；当前漏 x / 最大连挂 x`

## 6. 闯三关信号规则

闯三关重点不是“有没有号码”，而是“是否进入关卡动作”。

| 条件 | 等级 |
|---|---|
| 数据未加载或公式计算失败 | 暂停 |
| 当前已连续命中 2 关及以上 | 强观察 |
| 当前已过第 1 关 | 弱观察 |
| 未过第 1 关，但近期第三关通过率 >= 25% | 弱观察 |
| 未过第 1 关，且近期第三关通过率 < 25% | 暂停 |

首页卡片展示：

- 标题：`闯三关判断`
- 等级：`强观察 / 弱观察 / 暂停`
- 主文案：`当前动作：暂停 / 第一关观察 / 第二关观察 / 冲第三关`
- 说明：`本关号码池：xx xx；未中重置观察`

专项页顶部要强化三个判断：

| 判断项 | 展示 |
|---|---|
| 是否开第一关 | 是 / 否 |
| 是否续第二关 | 是 / 否 |
| 是否冲第三关 | 是 / 否 |

## 7. 世界杯信号规则

世界杯基于现有 `reliabilitySummary` 统计，不改比分模型。

| 条件 | 等级 |
|---|---|
| 世界杯数据未加载或数据源异常 | 暂停 |
| 可跟踪场次 >= 1 | 强观察 |
| 可跟踪为 0，但观察场次 >= 1 | 弱观察 |
| 可跟踪和观察都为 0，只有回避或无比赛 | 仅复盘 |

首页卡片展示：

- 标题：`世界杯比分`
- 等级：`强观察 / 弱观察 / 仅复盘 / 暂停`
- 主文案：`可跟踪 x 场 / 观察 y 场`
- 说明：`回避 z 场；最近完赛命中 x / 半中 y / 未中 z`

## 8. 首页信息架构调整

首页调整后顺序：

1. 今日总建议
2. 今日重点三张卡
3. 数据健康
4. 复盘摘要
5. 风险提醒

今日重点卡片增加等级标签：

| 元素 | 说明 |
|---|---|
| 模块标题 | 三中三推荐 / 闯三关判断 / 世界杯比分 |
| 信号等级 | 强观察 / 弱观察 / 仅复盘 / 暂停 |
| 主要动作 | 下期 6 码、当前关卡、可跟踪场次 |
| 解释 | 近期表现、号码池、回避场次 |
| 入口按钮 | 暂停时可进入，但按钮文案改为“查看原因” |

## 9. 非目标

- 不修改三中三号码生成公式。
- 不修改闯三关公式池。
- 不修改世界杯比分预测算法。
- 不新增新的采集源。
- 不承诺命中率，只做观察信号分层。

## 10. P0 实施范围

P0 只实现首页信号收敛：

1. 新增 `signalLevel` 统一字段。
2. 三个首页 summary 函数返回 `level`、`reason`。
3. 新增 `buildTodaySignalAdvice()`。
4. 首页渲染今日总建议。
5. 今日重点卡片显示信号等级。
6. 测试覆盖强制包含四类等级和总建议函数。

专项页内部的进一步收敛放到 P1。

## 11. 验收标准

1. 首页出现 `今日总建议`。
2. 首页三张重点卡都显示信号等级。
3. 页面源码包含四类等级：`强观察`、`弱观察`、`仅复盘`、`暂停`。
4. 三中三信号基于近 10 期命中、当前漏、最大连挂判断。
5. 闯三关信号基于当前关卡和近期第三关通过率判断。
6. 世界杯信号基于可跟踪、观察、回避和数据加载状态判断。
7. 数据异常时不显示强观察。
8. 现有菜单、世界杯、三中三相关测试继续通过。

## 12. 后续 P1

1. 三中三专项页顶部改成“推荐 / 风险 / 公式来源”三段式。
2. 闯三关专项页顶部改成“是否开第一关 / 是否续第二关 / 是否冲第三关”判断表。
3. 世界杯页面只默认展开可跟踪比赛，观察和回避折叠。
4. 数据与复盘页增加“仅复盘原因”入口。
